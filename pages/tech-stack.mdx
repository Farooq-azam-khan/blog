# Overall view of the Techstack

I use Python [FastAPI](https://fastapi.tiangolo.com/lo/) to develop apis. The benefits of [FastAPI](https://fastapi.tiangolo.com/lo/) are listed below:
*[Pydantic](https://docs.pydantic.dev/latest/) can be used to create typed routes
* RestAPI documentation is automatically generated throught the '/docs' routes
* modularity is easily achived, much like in `express` routes 
* asyn/await is supported out of the box 
* it is the perfect middle ground with easy route creation as well as the benefits of a typed system without the complex work on creating graphql routes. 
    * graphql types are much more robust 

I am also focused on data science and many python libraries are necessary for such project. Python FastAPI provides that convenience. 

When prototyping I do not care about Server Side Rendering (SSR); however, I do care a lot about a robust type system on the front end. [Elm](https://elm-lang.org/) is a compile-to-js language that adopts a functional programming paridgm. 

It is strongly typed and is extremely delightful to work in. The error messages are easy to follow and the [Elm](https://elm-lang.org/) Architecture is fast, secure, and reliable. [Elm](https://elm-lang.org/) is also batteries inclued, in that there is minimal configuration needed when starting a new project. For those reasons I chose [Elm](https://elm-lang.org/) over typescript, flow, and jsdocs. 

Lastly, I use [Vite](https://vitejs.dev/) as my bundler and [TailwindCSS](https://tailwindcss.com/) for styling. 

You can checkout my starter template [here](https://github.com/Farooq-azam-khan/fastapi-elm-tailwind-techstack).
